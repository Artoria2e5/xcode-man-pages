NAME

linkguard_blacklist —
protect a process from unwanted dynamic linkage

SYNOPSIS

#include <linkguard.h>

linkguard_blacklist(const char *name);

DESCRIPTION

Linkage graphs naturally grow in complexity. linkguard_blacklist provides a mechanism for restricting this growth.

The name argument is a the filename (specifically the last path component) of a dynamic library that you do not want to link. e.g. "UIKit", "libswiftCore.dylib", etc. name must be a C string literal.

linkguard_blacklist does not have to be called inside of a function scope. Regardless of where it is used, no executable code will be added to your binary. The blacklisted image names are stored in a new constant data section, called __linkguard. The blacklist is automatically enforced at runtime by liblinkguard.dylib, whenever dyld(1) loads new images.

When a blacklisted image is loaded, os_log_fault(3) is called with the reason ``Blacklisted library [...] was loaded.'', generating a crash report.

linkguard_blacklist has no behavioral impact on customer installs, because liblinkguard.dylib is not mastered into customer builds. This is why it must be weak linked.

liblinkguard.dylib can be injected into an already running process to test a blacklist without needing to relink the binary. See LINKGUARD_BLACKLIST.

linkguard_blacklist is disabled when processes are launched by Xcode.

CAVEATS

linkguard_blacklist is only available to executables.
Libraries cannot use linkguard_blacklist.

liblinkguard.dylib must be linked weakly.

Filenames are not entirely unambiguous ways to refer to linked images. If you end up in a world where you are trying to blacklist one of many libraries sharing the same filename, may god have mercy on your soul.

ENVIRONMENT

LINKGUARD_BLACKLIST
This environment variable overrides any existing blacklist in the process. When set, it is enforced as the one true blacklist. It should contain a comma separated list of image filenames. 
(e.g. "UIKit,libswiftCore.dylib")

This environment variable can be changed at any time, as it is checked when new images are loaded.

LINKGUARD_DISABLE
If this environment variable is set to anything, linkguard_blacklist will have no effect.

This variable must be inherited from the environment, and will not work if set at runtime.

INSPECTION

Since the blacklist is stored in the __linkguard section of the binary, it is quite easy to extract the NULL delimited blacklist from a binary using objdump(1): 
% objdump -j __linkguard -s example_binary
example_binary:	file format Mach-O 64-bit x86-64

Contents of section __linkguard:
 100001000 00000000 00000000 55494b69 74006c69  ........UIKit.li
 100001010 62737769 6674436f 72652e64 796c6962  bswiftCore.dylib
 100001020 00                                   .

The first 8 bytes (4 bytes on 32-bit) are part of a mechanism that ensures liblinkguard.dylib is only ever weak linked.
